# Bug Log — EggGrade OMS UAT

A living list of known bugs, risks, and limitations.

**How to use this file:**
- Add a new row whenever a bug or risk is discovered.
- Move closed rows to the "Closed" section at the bottom (keep them — historical context matters).
- Severity scale:
  - **🔴 Blocker** — data loss / app crash / operator cannot do their work
  - **🟠 High** — feature broken or seriously misleading; workaround required
  - **🟡 Medium** — degraded behavior, recovery possible without data loss
  - **🟢 Low** — cosmetic, documentation, or pre-existing known limitation
- Status scale: `open`, `investigating`, `workaround-in-place`, `fixed-not-yet-released`, `closed`.

This file replaces the previously-frozen `docs/KNOWN_ISSUES.md` (last updated 2026-05-09, archived as-is — no entries copied forward verbatim because they belong to an earlier code state).

## Severity philosophy

- 🔴 Blockers stop everything until fixed or rolled back
- 🟠 High issues require a workaround documented above and a near-term fix
- 🟡 Medium issues are acceptable in production but earn a place on the next stabilization sprint
- 🟢 Low issues are tracked for transparency but don't gate any build

If a row's severity changes, append a note in the row rather than rewriting — history matters when chasing regressions later.
